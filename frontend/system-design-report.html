1 Executive Summary

eHealth Malawi is a modular, AI-ready digital health platform designed for the Malawi Ministry of Health. It serves as both a National Digital Health Command Center and a comprehensive Hospital Management Information System (HMIS). The platform integrates patient management, clinical workflows, pharmacy, laboratory, billing, inventory, governance, critical care, dialysis, and a real-time national surveillance dashboard with an interactive district-level map of Malawi.

Design
Three-tier architecture (Frontend → API → Database) with offline-first configuration
Modules
30+ database tables, 35+ REST endpoints, 1 WebSocket, 18 user roles
Deployment
Dockerized FastAPI backend with SQLite; designed for district-level deployment

Key Architectural Decisions

1. Offline-First Design

Configured with OFFLINE_MODE=True by default, the system operates reliably in areas with intermittent connectivity. SQLite allows zero-configuration local deployment.

2. Modular Plugin Architecture

Each domain (clinical, pharmacy, lab, etc.) has isolated model, schema, and endpoint files. New modules can be added without touching existing code.

3. Database-Agnostic ORM

SQLAlchemy async ORM supports SQLite (default) and PostgreSQL. Configuration via a single DATABASE_URL environment variable.

4. AI-Ready Infrastructure

OpenAI SDK included in dependencies. Provider abstraction layer allows plugging in any AI service for clinical decision support and predictive analytics.

Entity-Relationship Highlights

  • Patient → Visit → Clinical Notes: One patient has many visits; each visit has one or more clinical/nursing notes and vital sign records.
  • Patient → Lab Orders → Lab Tests: One lab order contains multiple tests; results can be quantitative or qualitative with reference ranges.
  • Patient → Prescriptions → Prescription Items: Each prescription has multiple line items; administration records track actual dosage given.
  • Patient → Bills: Billing is patient-centric with support for partial payments and receipt references.
  • Visit → Queue: Queue tracks triage, consultation, pharmacy, lab, and procedure stages with position management.
  • User → Audit Trail: All data modifications are logged with user, action, resource, IP address, and timestamp.

7 Authentication & Security

JWT Bearer

All authenticated endpoints require Authorization: Bearer <token>. Tokens are HS256-signed with an 8-hour expiry. The get_current_user dependency decodes the token and loads the user from the database.

bcrypt Hashing

Passwords are hashed using bcrypt via the passlib library. The pwd_context utility provides hash() and verify() methods.

18 User Roles

Granular role-based access control: super_admin, hospital_admin, dho, doctor, nurse, clinical_officer, pharmacist, lab_technician, hsa, data_entry, etc.

Audit Trail

All data modifications are logged to the audit_trails and audit_logs tables with user ID, action, resource, IP address, and timestamp.

8 Frontend Architecture

The frontend is a single-page application (SPA) served entirely from one HTML file (frontend/index.html) with no framework dependencies — pure vanilla JavaScript, CSS, and HTML. It communicates with the backend exclusively through REST API calls (fetch()) and a WebSocket connection for real-time updates.

UI Architecture

Pages & Views

17 navigable views: MUST Clinic, Command Center, Patient Registry, Appointments, Queue, Triage, Clinical Notes, Vital Signs, Inpatient Ward, ICU, Dialysis, Lab, Pharmacy, Inventory, Billing, Governance, Admin.

NHISC Header

National Health Information & Search Center — fixed header with scrolling announcement banner, universal search, and user profile menu.

Interactive SVG Map

29 district paths of Malawi with choropleth coloring by indicator (births, outbreaks, facility status, referrals, stock-outs). Hover shows district name and value.

WebSocket Live Updates

Dashboard receives real-time pushes every 10 seconds for ops center cards, disease surveillance data, and AI-powered alerts.

10 Deployment

Docker Configuration

# Dockerfile
FROM python:3.11-slim
WORKDIR /app
COPY backend/requirements.txt .
RUN pip install -r requirements.txt
COPY backend/ .
CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8000"]

# docker-compose.yml
version: '3.8'
services:
  backend:
    build: .
    ports:
      - "8000:8000"
    volumes:
      - ./backend:/app
      - ehealth_data:/app
    environment:
      - DATABASE_URL=sqlite+aiosqlite:///./ehealth_malawi.db
      - OFFLINE_MODE=True
      - DEBUG=False
    restart: unless-stopped

Startup Sequence

  1. run.py calls create_app() from app.main
  2. App factory initializes CORS middleware, static mounts, lifespan handler
  3. Lifespan handler calls init_db() → Base.metadata.create_all() (auto-creates all tables)
  4. API routers are registered under /api/v1
  5. Static files (frontend, audio, uploads) are mounted
  6. Root route GET / serves index.html
  7. Server starts on 0.0.0.0:8000
  8. WebSocket endpoint is available for dashboard connections

12 Modules & Features

Patient Registry

Create, search, edit patients. Auto-generates EMW-XXXXXXX passport IDs. Stores demographics, allergies, diagnoses, emergency contacts.

Clinical Workflow

Visit management (OPD/IPD/ER/triage), SOAP clinical notes, nursing notes, vital signs with auto-BMI calculation, GCS scoring.

Appointments & Queue

Schedule, confirm, complete appointments. Real-time queue with position tracking, call buttons, department filtering, teleconsult flag.

Laboratory

Order lab tests, track specimen collection, record results with reference ranges and abnormal flags. Radiology order support with findings/reports.

Pharmacy

Prescription management with dosage tracking, controlled substance handling, dispensing workflow, drug stock alerts.

Dialysis & CKD

Hemodialysis/peritoneal session tracking, CKD staging (1–ESRD), eGFR monitoring, AV fistula management.

Critical Care (ICU)

ICU/HDU/NICU/PICU admissions, ventilation modes, SOFA/APACHE scoring, vasopressor tracking, fluid balance.

Inventory & Equipment

Stock management with reorder levels, lot tracking, expiry management, equipment maintenance scheduling.

Governance & Quality

Incident reporting (medication errors, falls, HAI, surgical), mortality reviews, quality indicators, full audit logging.

Billing & Payments

Bill creation, MPamba/Airtel Money/Bank payment processing, receipt generation, payment history.

National Dashboard

Real-time KPIs, ops center, disease surveillance, interactive district map (29 districts, 5 indicators), AI predictive insights, trend analytics.

Info Center

Announcements (critical/public health/service/admin), health intelligence summaries, universal search across patients and announcements.
